# フロントエンド実装計画

## 目的

traQ のアクティビティを「7,000 ノードの 3D 宇宙空間と光の波紋」として可視化する。
Three.js を採用し、60fps の維持と堅牢性を重視して実装を進める。

## フェーズ 1: プロジェクト基盤と 3 層アーキテクチャの構築

Vue のリアクティビティによるパフォーマンス劣化を防ぐため、「ロジック」「バッファ」「描画」を分離した基盤を作る。

### タスク 1.1: 初期設定と Three.js の導入

- Vue 3 と Vite の開発環境を構築する。
- Three.js を導入し、シーン、カメラ、レンダラーの基本構成を作成する。
- Retina ディスプレイなどで負荷が増えすぎないよう、レンダラーのピクセル比は 1.5 程度を上限に制限する。

### タスク 1.2: クライアントサイド・データストアの構築

- ロジック層として、7,000 ノードの親子関係やスコアを純粋なクラス配列 `ChannelGraph` として定義する。
- バッファ層として、Three.js の `InstancedMesh` に直結させる `Float32Array` バッファを用意する。
- UI 層として、選択中のチャンネル情報など最小限のデータだけを Vue の `reactive` / `ref` で管理する。

### タスク 1.3: SSE 通信と再接続・同期戦略の実装

- `EventSource` を用いたバックエンドとの接続機構を実装し、インパルスと定期同期を受信する。
- 通信断に備え、指数的バックオフによる自動再接続ロジックを実装する。
- 再接続成功時にはサーバーにフルスナップショットを要求し、フロントエンドの状態ずれをリセットする。

## フェーズ 2: 3D トポロジー配置と静的描画エンジンの実装

7,000 のチャンネルを重なりを抑えつつ有機的な星雲のように立体配置し、一括で描画する。

### タスク 2.1: 黄金角・単一ツリー 3D 配置アルゴリズムの実装

- Grand Root を中心に、全ノードを 1 本の巨大なツリーと見なす再帰関数を実装する。
- 黄金角（約 137.5 度）を利用して子ノードを螺旋状に配置し、階層 `depth` に応じて広がり角度と距離を減衰させる。
- リロードのたびに形状が変わるよう、配置開始角度と距離にランダムな揺らぎ（ジッター）を加える。
- UI フリーズを防ぐため、この計算は Web Worker で非同期に行うか、数フレームに分割して実行する。

### タスク 2.2: `InstancedMesh` による超高速描画とポストプロセス

- 7,000 ノードを 1 回のドローコールで描画するため、Three.js の `InstancedMesh` をセットアップする。
- 描画の遅延を防ぐため、ローディング画面の裏側で `renderer.compile()` を実行し、シェーダーを事前コンパイルする。
- `EffectComposer` と `UnrealBloomPass` を導入し、星や波紋が美しく発光する宇宙空間のルック&フィールを設定する。

### タスク 2.3: カメラワーク・Raycasting と UI 分離

- `OrbitControls` によるズーム、パン、回転を実装する。
- Vue コンポーネント上での操作がカメラ操作と競合しないよう、イベントの伝播制御を徹底する。
- マウスホバー / クリック判定（Raycasting）の負荷を下げるため、イベントをスロットリングし、`three-mesh-bvh` の導入を検討する。

## フェーズ 3: 動的エフェクトと自律減衰のパイプライン実装

60fps のメインループ内で、サーバーデータとの同期、自律減衰、光のアニメーション連鎖を制御する。

### タスク 3.1: 60fps メインループと同期パイプラインの構築

- `requestAnimationFrame` 内で、「自律減衰（指数減衰）」「サーバー値への補間（Lerp）」「バッファ更新」「GPU 転送（`needsUpdate`）」の順で実行されるパイプラインを構築する。
- タブが裏側に回った際の状態不整合を防ぐため、Page Visibility API を監視し、非アクティブ時のアニメーション停止と復帰時の状態スナップ（即時上書き）を実装する。

### タスク 3.2: 投稿イベント（降下粒子 → 波紋 → 伝播）の実装

- ルートから対象への降下粒子アニメーションと、着弾時の波紋（Ripple）エフェクトを実装する。
- 着弾時に、親ノードへの距離と速度から到達時間を計算し、タイマーで親の発熱と新たな波紋発生を再帰的にトリガーする。
- メモリリークと GC スパイクを防ぐため、波紋メッシュは毎回 `new` せず、オブジェクトプールから再利用する設計にする。

### タスク 3.3: 閲覧イベント（移動ビーム）の実装

- ノイズとしての控えめな光の線（トレイル）アニメーションを実装する。
- これらも同様にオブジェクトプールで管理し、生成・破棄のコストを最小化する。

### タスク 3.4: フィーバー状態の演出実装

- スコアが上限に達したチャンネルに対し、計算式にサイン波（`Math.sin`）を加えてサイズや輝度を脈動させるアニメーションを適用する。

### タスク 3.5: チャンネルの表示オプション

- 全チャンネル表示か、盛り上がり度が正のチャンネル表示を切り替える
- フルパスを表示する
- チャンネルを選び、そのがズームされるようにする
- クリックでチャンネルを選び、盛り上がり度などの詳細を見れるようにする
- 音楽もつける。光の線が飛ぶ時と、フィーバー状態のチャンネルが現れた時に効果音を流す。デフォルトで後ろで音楽も流すようにする。音楽は別で作る。

## フェーズ 4: エッジケース対応と最終チューニング

実環境特有のトラブルに対応し、本番リリースに向けた安定性を確立する。

### タスク 4.1: WebGL コンテキスト消失対策

- キャンバスの `webglcontextlost` イベントを監視し、GPU メモリ不足などで描画がクラッシュした場合に備える。
- Vue 側で適切なエラー表示とリロードを促すフォールバック UI を実装する。

### タスク 4.2: ブラウザリサイズへの追従

- ウィンドウリサイズやスマホの画面回転時に、カメラのアスペクト比とレンダラーのサイズを正しく再計算する処理を実装する。

### タスク 4.3: 負荷テストとパラメータチューニング

- ダミーの大量イベント（700 人アクティブ相当）を発生させる。
- 減衰率、Lerp 速度、Bloom の強度、オブジェクトプールのサイズを調整し、表現の美しさと 60fps 維持のバランスを最終確定する。
