# タスク【フロントエンド】


## プロジェクト基盤と 3 層アーキテクチャの構築

Vue のリアクティビティによるパフォーマンス劣化を防ぐため、「ロジック」「バッファ」「描画」を分離した基盤を作る。

### 初期設定と Three.js の導入

- 1: Three.js を導入し、シーン、カメラ、レンダラーの基本構成を作成する。
- 2: Retina ディスプレイなどで負荷が増えすぎないよう、レンダラーのピクセル比は 1.5 程度を上限に制限する。

### クライアントサイド・データストアの構築

- 3: ロジック層として、7,000 ノードの親子関係やスコアを純粋なクラス配列 `ChannelGraph` として定義する。
  - `ChannelGraph`: 純粋な TypeScript クラスとして実装するロジック層である。7,000ノードの親子関係、スコア計算、波紋の連鎖ロジックなど、毎フレームの重い計算を担当する。Vue のリアクティブシステムからは完全に切り離す。

```ts
class ChannelGraph {
  nodes: ChannelNode[] = [];
  nodeMap: Map<string, number> = new Map(); // ID -> index の高速ルックアップ用

  constructor(initData: any[]) {
    // サーバーからの初期データ（init）をパースし、7000件の nodes を構築する
  }

  // イベント処理

  // 投稿インパルスの受信
  triggerPostEvent(targetId: string) {
    const index = this.nodeMap.get(targetId);
    if (index === undefined) return;

    // 対象ノードのスコアをスパイクさせる
    this.nodes[index].currentScore += 50;

    // 親への「波紋連鎖タイマー」をイベントキューに登録する
    this.scheduleRippleToParent(this.nodes[index]);
  }

  // サーバーからの定期同期（activity_update）の受信
  syncServerScores(updates: { id: string; score: number }[]) {
    for (const update of updates) {
      const index = this.nodeMap.get(update.id);
      if (index !== undefined) {
        // 現在値は上書きせず、目標値（targetScore）のみを更新する
        this.nodes[index].targetScore = update.score;
      }
    }
  }
}
```

- 4: バッファ層として、Three.js の `InstancedMesh` に直結させる `NodeBuffer` バッファを用意する。
  - `NodeBuffer`: `Float32Array` を使うバッファ層である。ChannelGraph で計算した結果を GPU（Three.js）へ転送するための一次元配列として扱う。座標、サイズ、色などの描画用データを保持する。

```ts
class NodeBuffer {
  // 1インスタンスあたり 16個の Float 値（Matrix4）
  matrixData: Float32Array;
  // 1インスタンスあたり 3個の Float 値（R, G, B）
  colorData: Float32Array;

  constructor(count: number) {
    this.matrixData = new Float32Array(count * 16);
    this.colorData = new Float32Array(count * 3);
  }

  // 初期の3D座標をセットする（静的配置タスク 2.1 用）
  setPosition(index: number, x: number, y: number, z: number) {
    // matrixData の該当インデックスに平行移動をセットする
  }
}
```

- 5: UI 層``AppState`を作る。選択中のチャンネル情報など最小限のデータだけを Vue の `reactive` / `ref` で管理する。
  - Vue の `ref` や `reactive` で管理する UI 層である。現在ユーザーがクリックして選択しているチャンネルの ID や名前、全体のローディング状態など、UI の描画に必要な最小限のデータだけを持つ。

### インターフェース設計
```ts
interface ChannelNode {
  index: number;         // 0~6999 の固定インデックス（配列アクセス用・最速）
  id: string;            // UUID（API との照合用）
  parentId: string | null;
  children: number[];    // 子ノードのインデックス配列

  islandId: number;      // 属する島の ID（0~8）
  depth: number;         // ルートからの深さ（色や配置の計算に使用）

  // 状態（スコア）
  currentScore: number;   // 現在の盛り上がり度（描画に直結）
  targetScore: number;    // サーバーから同期された目標スコア（Lerp 用）
}
```

### SSE 通信と再接続・同期戦略の実装

- 6: `EventSource` を用いたバックエンドとの接続機構を実装し、インパルスと定期同期を受信する。
- 7: 通信断に備え、指数的バックオフによる自動再接続ロジックを実装する。
- 8: 再接続成功時にはサーバーにフルスナップショットを要求し、フロントエンドの状態ずれをリセットする。

## 3D トポロジー配置と静的描画エンジンの実装

7,000 のチャンネルを重なりを抑えつつ有機的な星雲のように立体配置し、一括で描画する。

### 黄金角・単一ツリー 3D 配置アルゴリズムの実装

- 9: Grand Root を中心に、全ノードを 1 本の巨大なツリーと見なす再帰関数を実装する。
- 10: 黄金角（約 137.5 度）を利用して子ノードを螺旋状に配置し、階層 `depth` に応じて広がり角度と距離を減衰させる。
- 11: リロードのたびに形状が変わるよう、配置開始角度と距離にランダムな揺らぎ（ジッター）を加える。
- 12: UI フリーズを防ぐため、この計算は Web Worker で非同期に行うか、数フレームに分割して実行する。

### `InstancedMesh` による超高速描画とポストプロセス

- 13: 7,000 ノードを 1 回のドローコールで描画するため、Three.js の `InstancedMesh` をセットアップする。
- 14: 描画の遅延を防ぐため、ローディング画面の裏側で `renderer.compile()` を実行し、シェーダーを事前コンパイルする。
- 15: `EffectComposer` と `UnrealBloomPass` を導入し、星や波紋が美しく発光する宇宙空間のルック&フィールを設定する。

### カメラワーク・Raycasting と UI 分離

- 16: `OrbitControls` によるズーム、パン、回転を実装する。
- 17: Vue コンポーネント上での操作がカメラ操作と競合しないよう、イベントの伝播制御を徹底する。
- 18: マウスホバー / クリック判定（Raycasting）の負荷を下げるため、イベントをスロットリングし、`three-mesh-bvh` の導入を検討する。

## 動的エフェクトと自律減衰のパイプライン実装

60fps のメインループ内で、サーバーデータとの同期、自律減衰、光のアニメーション連鎖を制御する。

### 60fps メインループと同期パイプラインの構築

- 19: `requestAnimationFrame` 内で、「自律減衰（指数減衰）」「サーバー値への補間（Lerp）」「バッファ更新」「GPU 転送（`needsUpdate`）」の順で実行されるパイプラインを構築する。
- 20: タブが裏側に回った際の状態不整合を防ぐため、Page Visibility API を監視し、非アクティブ時のアニメーション停止と復帰時の状態スナップ（即時上書き）を実装する。

### 投稿イベント（降下粒子 → 波紋 → 伝播）の実装

- 21: ルートから対象への降下粒子アニメーションと、着弾時の波紋（Ripple）エフェクトを実装する。
- 22: 着弾時に、親ノードへの距離と速度から到達時間を計算し、タイマーで親の発熱と新たな波紋発生を再帰的にトリガーする。
- 23: メモリリークと GC スパイクを防ぐため、波紋メッシュは毎回 `new` せず、オブジェクトプールから再利用する設計にする。

###  閲覧イベント（移動ビーム）の実装

- 24: ノイズとしての控えめな光の線（トレイル）アニメーションを実装する。
- 25: これらも同様にオブジェクトプールで管理し、生成・破棄のコストを最小化する。

### フィーバー状態の演出実装

- 26: スコアが上限に達したチャンネルに対し、計算式にサイン波（`Math.sin`）を加えてサイズや輝度を脈動させるアニメーションを適用する。

### チャンネルの表示オプション

- 27: 全チャンネル表示か、盛り上がり度が正のチャンネル表示を切り替える
- 28: フルパスを表示出来るようにする
- 29: チャンネルを選び、そのがズームされるようにする
- 30: クリックでチャンネルを選び、盛り上がり度などの詳細を見れるようにする
- 31: 音楽もつける。光の線が飛ぶ時と、フィーバー状態のチャンネルが現れた時に効果音を流す。デフォルトで後ろで音楽も流すようにする。音楽は別で作る。

## フエッジケース対応と最終チューニング

実環境特有のトラブルに対応し、本番リリースに向けた安定性を確立する。

### WebGL コンテキスト消失対策

- 32: キャンバスの `webglcontextlost` イベントを監視し、GPU メモリ不足などで描画がクラッシュした場合に備える。
- 33: Vue 側で適切なエラー表示とリロードを促すフォールバック UI を実装する。

### ブラウザリサイズへの追従

- 34: ウィンドウリサイズやスマホの画面回転時に、カメラのアスペクト比とレンダラーのサイズを正しく再計算する処理を実装する。

### 負荷テストとパラメータチューニング

- 35: ダミーの大量イベント（700 人アクティブ相当）を発生させる。
- 36: 減衰率、Lerp 速度、Bloom の強度、オブジェクトプールのサイズを調整し、表現の美しさと 60fps 維持のバランスを最終確定する。
